Problem Statement: Hotel Review Intelligence Engine 
Build an AI-powered hotel intelligence system that analyzes hotel reviews and ratings to assess hotel performance over time, detect seasonal and aspect-level sentiment shifts, and generate personalized, evidence-based hotel recommendations based on user profile, review context, and semantic intent. 

Examples of what the solution may include: 
•	Hotel performance tracking over time using reviews and ratings 
•	Aspect-based reasoning across themes such as cleanliness, service, amenities, value, or location 
•	Detection of seasonal or temporal sentiment drift 
•	Personalized hotel recommendations based on user profile 
•	Contradiction handling across different types of reviewers 
•	Semantic search, retrieval, or RAG-based recommendation flows 

What strong solutions may demonstrate: 
•	Intelligent processing of structured and unstructured review data 
•	Temporal reasoning and trend detection 
•	Profile-aware recommendation logic 
•	Semantic retrieval and evidence-based summarization 
•	Strong backend workflow or pipeline design 
•	Actionable outputs instead of generic summaries 

Suggested tools for the Hotel Review Intelligence: 
•	Python 
•	Pandas and NumPy 
•	Open-source LLMs 
•	Hugging Face transformers 
•	spaCy, NLTK, or sentence-transformers 
•	FAISS or Chroma for local retrieval if needed 
•	Plotly, Matplotlib, or Seaborn 
•	Streamlit or Gradio for demos 
•	backend frameworks or data processing pipelines as needed

Common FAQs 
1. Can we participate individually or only in teams?
Participants must participate individually. Team participation is not allowed.

2. Can we use open-source tools and models?
Yes. Participants may use free and open-source tools, frameworks, libraries, and models, provided they comply with the event rules and applicable usage terms.

3. Can we use external APIs or public datasets?
Participants should primarily use the datasets shared as part of the hackathon toolkit. Use of additional public datasets or external APIs should be treated as optional

4. Do we need to build a fully production-ready solution?
No. The solution does not need to be production-ready, but it should be logically sound, relevant to the problem statement, and clearly demonstrated.

5. Is a polished UI mandatory?
No. A polished interface is not mandatory. Teams may submit a chatbot, notebook, dashboard, web app, or any other working prototype, as long as the solution is clear and usable.

6. Can we make assumptions if the dataset is incomplete?
Yes. Individuals may make reasonable assumptions where needed, but they should clearly document those assumptions in their submission.

7. What are the mandatory deliverables?
Participants are expected to submit a solution summary, presentation deck, working prototype or proof of concept, demo recording if required, code repository or project files, and a short README or documentation note.

8. How will submissions be judged?
Submissions will be evaluated on relevance to the problem, strength of solution design, technical depth, quality of demonstration, impact potential, and clarity of communication.

9. Can we combine multiple use cases within the same problem statement?
Yes. Participants may focus on one core use case or combine multiple connected use cases, depending on the depth and scope of their solution.

10. What happens if a demo link or submission link does not work?
Participants are responsible for ensuring that all links are accessible and working before the deadline. Non-working links may affect evaluation.


Toolkit-Specific FAQs — Problem Statement 2
Hotel Review Intelligence Engine
1. Do we need to analyze all reviews in the dataset?
Yes, Participants may work with the full dataset or a representative subset for prototyping, as long as the solution approach is clearly demonstrated.

2. Is seasonal or temporal sentiment drift mandatory?
It is an important part of the challenge and Participants are encouraged to address it, especially if their solution focuses on hotel performance over time.

3. Are we expected to personalize recommendations using user profile data?
Yes, personalization is a core part of the problem statement and participants are encouraged to show how user context influences recommendations or summaries.

4. Do we need to explicitly handle contradictory reviews?
If relevant to the proposed approach, yes. Contradictory reviews are a realistic challenge and handling them thoughtfully can strengthen the solution.

5. Is RAG required for this problem statement?
No. Retrieval-augmented generation is one possible approach, but not a requirement. Teams may use any suitable method to generate evidence-based outputs.

6. What kind of output is expected?
The model should return the top 5 recommendations, ranked by relevance (personalized hotel recommendations, hotel insight summaries, aspect-based hotel scorecards, dashboards, or other formats that clearly demonstrate value). Each recommendation should follow the defined output schema, including all required fields as specified in the sample output provided for validation.

7. Do we need aspect-level analysis such as cleanliness, service, and amenities?
This is not strictly mandatory, but aspect-level reasoning is highly relevant and can make the solution stronger and more actionable.

8. Can we use open-source NLP models, embeddings, or vector search tools?
Yes. Teams may use open-source NLP libraries, embeddings, retrieval systems, and other free or open-source components as needed.
